# Прогноз опадів на основі Open-Meteo

Streamlit-застосунок, який завантажує реальні метеодані, навчає модель
класифікації та прогнозує наявність опадів для обраного дня.

## Які дані використовуються

Дані завантажуються з безкоштовного сервісу [**Open-Meteo**](https://open-meteo.com)
через HTTP-запит (щоденні дані, `daily`).
|Поле|Опис|
|-|-|
|`precipitation\_sum`|Сума опадів за день (мм) — **цільова змінна**|
|`rain\_sum`|Сума дощу за день (мм) — ознака|
|`temperature\_2m\_max`|Максимальна температура (°C)|
|`temperature\_2m\_min`|Мінімальна температура (°C)|
|`temperature\_2m\_mean`|Середня температура (°C)|
|`windspeed\_10m\_max`|Максимальна швидкість вітру (км/год)|
|`shortwave\_radiation\_sum`|Сумарна сонячна радіація (МДж/м²)|
|`et0\_fao\_evapotranspiration`|Евапотранспірація (мм)|

**Цільова змінна:**
* `0` — опадів немає (`precipitation\_sum == 0`)
* `1` — опади є (`precipitation\_sum > 0`)

За замовчуванням місто — **Київ** (lat=50.4501, lon=30.5234).
Рекомендований обсяг даних — від 1 року.

## Які моделі використовуються

|Модель|Бібліотека|Опис|
|-|-|-|
|**Random Forest**|scikit-learn|Ансамблева модель дерев рішень (200 дерев). Показує важливість ознак.|
|**Logistic Regression**|scikit-learn|Лінійна модель, дані масштабуються `StandardScaler`.|

Можна обрати одну модель або навчити обидві одразу.
Для фінального прогнозу автоматично обирається модель із найвищим F1-score на тестовому наборі.

### Метрики оцінки
* **Accuracy** — частка правильних прогнозів
* **Precision** — з усіх передбачених «є опади», скільки справжніх
* **Recall** — з усіх реальних «є опади», скільки виявлено
* **F1-score** — гармонійне середнє Precision та Recall

## Структура проєкту

```
weather\_project/
│
├── app.py               # Головний файл Streamlit-застосунку
├── weather\_daily.csv    # Датасет (генерується автоматично або завантажується вручну)
├── requirements.txt     # Залежності Python
└── README.md            # Цей файл
```
### Послідовність дій у застосунку
1. **Блок 1 — Дані**
   * Натисніть «Отримати дані з Open-Meteo» (або завантажте CSV).
   * Перегляньте статистику та графіки розподілу.
   * Збережіть датасет кнопкою «Скачати CSV».
2. **Блок 2 — Навчання моделі**
   * Оберіть алгоритм у бічній панелі (Random Forest / Logistic Regression / обидві).
   * Натисніть «Навчити модель».
   * Перегляньте метрики, матрицю плутанини та важливість ознак.
3. **Блок 3 — Прогноз**
   * Оберіть день зі списку.
   * Натисніть «Зробити прогноз».
   * Отримайте результат: «Очікуються опади» / «Опадів не очікується» та ймовірність.

## Запуск
```bash
pip install -r requirements.txt
streamlit run app.py
```
Застосунок відкриється у браузері за адресою `http://localhost:8501`.
